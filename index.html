Installation

Three lines to augment.

// 1. Wrap your app
import { GhostLens } from 'ghost-lens';

export default function App() {
  return (
    <GhostLens idleDelay={3000}>
      <YourApp />
    </GhostLens>
  );
}

// 2. Add data-ghost to any element
<button data-ghost="Click to save your changes to the cloud">
  Save
</button>

// 3. Or use the ghost() helper
import { ghost } from 'ghost-lens';

<button {...ghost('Click to save your changes')}>Save</button>
<!-- 1. Import the web component -->
<script type="module">
  import 'ghost-lens/web-component';
</script>

<!-- 2. Drop the element anywhere -->
<ghost-lens idle-delay="3000"></ghost-lens>

<!-- 3. Add data-ghost attributes -->
<button data-ghost="Click to submit the form">
  Submit
</button>
import { GhostLensEngine } from 'ghost-lens';

// Initialize the engine
const lens = new GhostLensEngine(null, {
  idleDelay: 3000,
  onActivate: () => console.log('Ghost Lens active'),
});

// Add data-ghost to HTML elements
// <button data-ghost="Help text">Click me</button>

// Manual control
lens.activate();
lens.deactivate();
lens.destroy();
Why Ghost Lens

Help that earns its keep.

👁
Behavioral Trigger
Detects genuine confusion via idle time — not button clicks. Appears exactly when needed, disappears immediately on activity.
⚡
Zero Dependencies
Core engine is pure vanilla JS. React and Web Component wrappers are optional. ~3kb gzipped. Doesn't bloat your bundle.
🎯
Spatial Awareness
Tooltips calculate real-time bounding boxes of target elements. Smart viewport clamping means no tooltips escape off-screen.
🎨
Kinetic Reveal
The Clear Curtain sweep feels intentional — like a HUD powering up. Users notice it as a feature, not an intrusion.
🔌
Framework Agnostic
React hook, JSX component, Web Component, or raw engine. One SDK, every stack. Drop it in a WordPress site or a Next.js app.
🔒
Respects the User
"Disable Ghost Lens Forever" persists to localStorage. No dark patterns. Users who don't want it never see it again.
